# S/A/B/C 루브릭 및 LLM 출력 스키마

## 등급 정의

| 등급 | 의미 | 서류 단계 권고 |
|------|------|----------------|
| **S** | 핵심인력 | 즉시 심층 검토·면접 우선 검토 |
| **A** | 우수인력 | 적극 검토 |
| **B** | 평균인력 | 필요 시 추가 자료·보류 검토 |
| **C** | 불합격 권고 | 우선순위 낮음 (자동 불합격 아님) |

모든 등급은 **보조 의견**이며 최종 판단은 채용 담당자에게 있습니다.

## 평가 시 고려 요소

- 공고(JD)의 필수 자격·우대·역할과의 적합성
- 경력 깊이, 업종·직무 연관성
- 학력·스킬의 직무 정합성
- 이력서·소개의 구체성과 성과 서술
- 지원 경로만으로 등급을 바꾸지 말 것 (편향 완화)

## 부문별 그룹핑

- 기본 키: **`job_id`**
- Slack 섹션 제목: `{team_title} / {job_title}` (없으면 `job_title`만)

## LLM 출력 JSON 스키마

한 공고(`job_id`)당 하나의 객체:

```json
{
  "job_id": 4664,
  "job_title": "CFO(Chief Financial Officer)",
  "team_title": "경영지원실",
  "summary": "한 줄 요약",
  "grades": [
    {
      "candidate_id": 1001,
      "grade": "A",
      "reason": "JD의 OO 요건과 경력 N년이 정합됨.",
      "risks": "해당 산업 경험은 상대적으로 짧음."
    }
  ]
}
```

- `grade`는 반드시 `S`, `A`, `B`, `C` 중 하나.
- 모든 `applied` 후보에 대해 항목을 포함할 것(불확실하면 B/C와 사유 명시).
