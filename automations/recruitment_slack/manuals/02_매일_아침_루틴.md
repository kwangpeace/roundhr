# 02. 매일 아침 루틴 (3단계, 약 5분)

## 1단계. Cursor에서 판정 결과 받기

Cursor 채팅창에 아래 프롬프트를 **그대로 붙여넣고 Enter**.

> 라운드HR MCP로 `job_stage_kind == "applied"` 지원자를 다음 기간으로 조회해 주세요.  
> - 화~금: 전날 09:50 KST ~ 지금  
> - 월: 직전 금요일 10:00 KST ~ 지금  
>
> 그리고 각 지원자를 `automations/recruitment_slack/config/jd/{job_id}.txt` 의 JD와 평가지표에 맞춰 **S / A / B / C** 로 판정해 주세요.  
> - S: 핵심 인재, JD 강한 매치  
> - A: 좋은 매치, 소폭 갭  
> - B: 보통, 추가 검토 필요  
> - C: 서류 통과 비권장  
>
> 결과는 `automations/recruitment_slack/message.template.md` 와 **완전히 같은 마크다운 포맷**의 **한 덩어리**로 출력해 주세요. 공고별로 묶고, 등급은 S → A → B → C 순으로 정렬. JSON이나 추가 설명 문장은 쓰지 말아 주세요.

Cursor가 몇 초~몇십 초 뒤 한 번에 정리된 결과를 보여 줍니다.

## 2단계. 결과를 `message.md` 로 저장

1. Cursor 응답 **전체를 복사** (`Ctrl+A` → `Ctrl+C`)
2. `automations/recruitment_slack` 폴더에 **`message.md`** 를 새로 만들고 붙여넣기
3. 저장 (`Ctrl+S`)

> 이미 `message.md` 가 있으면 덮어써도 됩니다. 오늘 날짜 기준으로 새로 쓰는 개념이에요.

### 결과가 이상할 때

- 등급이 빠지거나, 근거가 부족하거나, 형식이 안 맞으면 **Cursor에 다시 요청**해서 받으세요.  
  예: “등급 순서대로 다시 정리해 주세요” / “홍길동 지원자의 근거를 더 구체적으로 써 주세요”
- 사람이 직접 고쳐도 됩니다. 자세한 건 [03_메시지_수정하기.md](03_메시지_수정하기.md).

## 3단계. `post_message.bat` 더블클릭

1. 탐색기에서 `automations/recruitment_slack/post_message.bat` 더블클릭
2. 검은 창이 뜨면서 Slack에 메시지가 전송됩니다
3. `[done] Slack 전송 성공.` 이 보이면 완료
4. 아무 키나 누르면 창 닫힘

Slack 채널(`지원자추천`)에 S → A → B → C 순으로 정리된 요약이 도착했는지 확인하세요.

## 자주 묻는 질문

### Q. 지원자가 0명이면 어떻게 하나요?
- Cursor가 “오늘 해당 기간 내 `applied` 지원자가 없습니다” 라고 답합니다.
- 그 문장도 그대로 `message.md` 에 넣고 실행하면 Slack에 같은 내용이 전송됩니다.
- 또는 그냥 **오늘은 실행 안 하기** 로 두셔도 됩니다.

### Q. Slack에 보내기 전에 내용만 확인하고 싶어요
- `post_message.bat` 대신 터미널에서:

  ```
  .\.venv\Scripts\python -m src.post_message -f message.md --dry-run
  ```

  → Slack에 안 가고 콘솔에만 출력.

### Q. 메시지를 잘못 보냈어요
- Slack에서 해당 메시지 오른쪽 위 “⋯” → **삭제**.
- 이 시스템에는 “되돌리기” 기능이 따로 없습니다.
