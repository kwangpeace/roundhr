# 04. 평가 기준 수정하기 (공고별 JD·평가지표)

공고가 새로 열렸거나, 기존 평가지표가 바뀌었을 때만 하시면 됩니다.

## 폴더 위치

```
automations/recruitment_slack/config/jd/
```

여기에 **공고 하나당 파일 하나** 가 있습니다.

현재 들어 있는 파일:

| 파일명 | 포지션 |
|--------|--------|
| `4664.txt` | CFO |
| `6001.txt` | 서비스 기획자 |
| `6002.txt` | AI 컨설턴트 |
| `6003.txt` | 연구소장 |
| `6004.txt` | R&D PM |

> 파일 이름의 숫자 = **라운드HR의 `job_id`** 입니다. 라운드HR에서 실제 ID를 확인한 뒤 맞춰 주세요.

## 1) 기존 공고 기준을 수정하고 싶을 때

1. `config/jd/` 안의 해당 `.txt` 파일을 메모장이나 Cursor로 엽니다.
2. 내용을 원하는 대로 수정하고 저장 (UTF-8).
3. 이후 Cursor에 판정 요청하면 **자동으로 새 기준이 반영**됩니다. 별도 명령 불필요.

## 2) 공고를 새로 추가하고 싶을 때

1. 라운드HR에서 해당 공고의 **`job_id`** 를 확인 (예: `7001`)
2. 같은 폴더에 **`7001.txt`** 파일 새로 만들기
3. 공고 JD와 평가지표를 통째로 붙여넣기 (구글 문서에서 복사해도 됨)
4. 저장. 끝.

## 3) 공고를 삭제하고 싶을 때

해당 파일을 삭제하거나, 확장자를 바꿔 두면 됩니다(예: `.txt` → `.txt.bak`).

## 4) 파일을 못 찾으면 어떻게 되나요?

해당 공고의 지원자는 **JD 없이** 공고 제목·팀 정보만 가지고 판정됩니다. 정확도가 떨어지므로 **공고가 늘어날 때마다 파일을 추가**하는 걸 권장합니다.

## 5) 평가 기준 작성 팁

- 공고 본문 + 평가지표 + 면접 질문을 한 파일에 다 넣어도 됩니다. Cursor는 긴 텍스트도 잘 읽습니다.
- **필수 자격(미충족 시 자동 탈락)** 부분을 굵게/눈에 띄게 해두면 판정 품질이 올라갑니다.
- **합격 컷오프/등급 정의**를 명시하면 더 일관된 등급이 나옵니다.

## 6) 변경 후 빠르게 검증하려면

1. 기준을 바꾸기 전에 받은 `message.md` 와  
2. 기준을 바꾼 뒤 새로 받은 `message.md` 를 비교

차이가 크면 기준이 너무 엄격하거나 느슨해진 것입니다. 문구를 조정하세요.

## 자주 하는 실수

- 파일명을 `4664.txt` 가 아니라 `4664` 로 저장 → 시스템이 JD로 인식하지 못함
- 파일을 `CFO.txt` 같은 한글·영문 이름으로 저장 → 마찬가지로 인식 못 함. **반드시 숫자(job_id).txt**
- 인코딩이 UTF-8이 아닌 ANSI → 한글 깨짐. 메모장 저장 시 "UTF-8" 선택
