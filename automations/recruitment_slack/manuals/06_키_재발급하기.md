# 06. 키 재발급하기 (보안)

Slack 토큰이나 Gemini 키가 외부(채팅, 깃허브, 문서)에 노출됐다면 **바로 재발급** 하세요. 악용을 막기 위한 기본 절차입니다.

## 언제 꼭 재발급해야 하나요?

- 채팅창, 이메일, 문서 등에 키를 붙여 넣어 **다른 사람에게 보이게** 된 경우
- 퇴사자 PC에 접근 기록이 있는 경우
- 이유는 모르지만 Slack·Gemini에서 **의심 알림**이 온 경우

## Slack 봇 토큰 재발급

1. [https://api.slack.com/apps](https://api.slack.com/apps) 접속
2. 해당 봇 앱 선택 → 왼쪽 **OAuth & Permissions**
3. **Bot User OAuth Token** 오른쪽의 **Regenerate** 또는 **Revoke** → 재설치
4. 새로 나온 `xoxb-...` 값을 복사
5. `automations/recruitment_slack/.env` 의 `SLACK_BOT_TOKEN` 을 새 값으로 교체
6. `post_message.bat` 한 번 실행해 정상 동작 확인

## Gemini API 키 재발급 (혹시 쓰는 경우)

1. [https://aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey) 접속
2. 기존 키 옆 휴지통 아이콘으로 **삭제**
3. **Create API key** 로 새 키 발급
4. `.env` 의 `LLM_API_KEY` 를 새 값으로 교체

> Cursor-only 운영이라면 Gemini 키는 빈 값이어도 괜찮습니다.

## `.env` 편집 시 주의

- **따옴표 없이** 값만 적습니다. 예: `SLACK_BOT_TOKEN=xoxb-...`
- 앞뒤에 공백이 들어가지 않도록 주의
- 저장 시 **인코딩은 UTF-8**

## 혹시 Git에 실수로 키를 올린 경우

`.env` 는 `.gitignore` 로 막혀 있어 기본적으로 올라가지 않습니다. 하지만 다른 파일에 키를 적어 올린 경우엔:

1. 즉시 **위 절차로 키 재발급**
2. 레포에서 해당 파일을 지우고 커밋/푸시
3. 이력까지 지워야 한다면 담당 엔지니어에게 요청(`git filter-repo` 등 필요)

## 끝

재발급 직후에는 반드시:

- `post_message.bat` 로 **메시지 1건** 전송 테스트
- Slack 채널에 메시지가 도착하면 OK
- 오류가 나면 [05_자주_발생하는_문제.md](05_자주_발생하는_문제.md) 참고
