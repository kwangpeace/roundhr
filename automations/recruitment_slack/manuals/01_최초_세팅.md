# 01. 최초 세팅 (한 PC에 한 번만)

이 PC에서 한 번 성공했다면 다시 할 필요 없어요. 다른 노트북 등에서 처음 시작할 때만 합니다.

## 준비물

- Windows PC
- **Python 3.12 이상** 설치 ([python.org](https://www.python.org/downloads/) → "Add to PATH" 체크)
- **Cursor**
- **Slack 워크스페이스**에 봇이 설치되어 있을 것
- 이 프로젝트 폴더가 PC에 있을 것  
  예: `C:\Users\admin\OneDrive\Desktop\커서\roundhr`

## 1) 폴더 열기

파일 탐색기에서:

```
C:\Users\admin\OneDrive\Desktop\커서\roundhr\automations\recruitment_slack
```

## 2) `.env` 파일 만들기

같은 폴더 안 `.env.example` 을 **복사해서** 이름을 `.env` 로 바꿉니다.

`.env` 파일을 메모장(또는 Cursor)으로 열고 아래 값을 채웁니다.

```
LLM_PROVIDER=gemini
LLM_API_KEY=
LLM_MODEL=gemini-2.5-flash
SLACK_BOT_TOKEN=xoxb-로 시작하는 값
SLACK_CHANNEL_ID=C로 시작하는 값
DRY_RUN=false
MOCK_LLM=false
```

- **`LLM_*` 부분은 비워둬도 됩니다.** (Cursor-only 운영이면 사용하지 않음)
- **`SLACK_BOT_TOKEN`, `SLACK_CHANNEL_ID` 이 두 개만 반드시 채우세요.**

## 3) Slack 봇을 채널에 초대

Slack에서 알림이 올 채널(예: `#지원자추천`)에 들어가서:

```
/invite @봇이름
```

을 입력하거나, 채널 설정 → **통합** → **앱 추가**로 봇을 채널에 추가합니다.

> 이걸 안 하면 첫 실행 시 `not_in_channel` 오류가 납니다.

## 4) 파이썬 환경 자동 준비 (처음 한 번)

`automations/recruitment_slack/post_message.bat` 을 **더블클릭**하세요.

- 처음이면 자동으로 `.venv` 폴더가 만들어지고 필요한 패키지가 설치됩니다.
- 그 다음 `message.md` 가 없다고 뜨면서 종료됩니다. → 정상입니다.

## 5) 최초 동작 테스트

같은 폴더의 `message.template.md` 를 **복사**해서 이름을 `message.md` 로 바꾼 뒤, `post_message.bat` 를 더블클릭합니다.

- Slack 채널에 예시 메시지가 도착하면 **성공**.
- 오류가 나면 [05_자주_발생하는_문제.md](05_자주_발생하는_문제.md) 를 보세요.
