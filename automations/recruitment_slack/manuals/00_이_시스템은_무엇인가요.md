# 00. 이 시스템은 무엇인가요?

## 목적 한 줄 요약

> **라운드HR로 들어온 서류 지원자를 매일 한 번씩 훑어보고, 합격 가능성이 높은 사람을 Slack으로 알려주는 시스템입니다.**

## 왜 필요한가요?

- 지원자가 매일 쏟아질 때 한 명 한 명 따로 보기 부담스러움
- 공고별로 평가 기준(JD, 평가지표)이 다른데, 매번 기준을 다시 훑기 번거로움
- 담당자에게 **“오늘의 서류합격 후보”**만 빠르게 공유하고 싶음

## 어떤 흐름으로 돌아가나요?

```
[라운드HR]  ─ MCP 조회 ─>  [Cursor AI]  ─ 판정 결과 ─>  [message.md]
                                                           │
                                                    post_message.bat
                                                           │
                                                           ▼
                                                        [Slack]
```

- **라운드HR**: 지원자 원본 데이터
- **Cursor AI**: 라운드HR MCP로 지원자를 조회하고, 공고별 JD·평가지표에 맞춰 **S/A/B/C** 등급을 매김
- **message.md**: Cursor가 정리해 준 결과를 사람이 확인·저장하는 파일
- **post_message.bat**: `message.md` 내용을 Slack 채널로 전송하는 실행 파일

## 등급 정의 (간단)

| 등급 | 뜻 | 후속 |
|------|----|------|
| **S** | 핵심 인재, JD 강한 매치 | 즉시 후속 전형 권장 |
| **A** | 좋은 매치, 소폭 갭 | 서류 합격 권장 |
| **B** | 보통, 추가 검토 필요 | 포트폴리오/면접에서 확인 |
| **C** | 서류 통과 비권장 | 담당자가 최종 판단 |

> S·A가 **"서류합격자, 평가 등급 우수자"** 에 해당합니다.

  `C:\Users\admin\OneDrive\Desktop\커서\roundhr\automations\recruitment_slack\`

## 무엇을 안 써도 되나요?

- **Gemini API 키** – 지금은 Cursor가 직접 판정하므로 없어도 됩니다.
- **GitHub Actions / Railway Cron** – Cursor로 수동 실행하므로 끄거나 무시해도 됩니다. (나중에 자동화가 필요할 때를 위해 코드는 남겨 둠)
