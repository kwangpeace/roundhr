# 00. 이 시스템은 무엇인가요?

## 목적 한 줄 요약

> **라운드HR에 들어온 서류(`applied`) 지원자를 기준에 맞춰 S/A/B/C로 정리하고, 그 요약을 Slack으로 보냅니다.**

## 왜 필요한가요?

- 지원자가 많을 때 공고별 기준을 매번 다시 읽기 번거로움
- 담당자에게 **오늘 볼 후보**만 빠르게 공유하고 싶음

## 흐름

```
[라운드HR]  ─ MCP (Cursor 안에서만) ─>  [Cursor AI]  ─ 결과 ─>  [message.md]
                                                                  │
                                                         send_slack.bat
                                                         (토큰|채널 한 줄)
                                                                  │
                                                                  ▼
                                                              [Slack]
```

- **라운드HR MCP**: Cursor 채팅에서만 호출됩니다. 일반 터미널만으로는 라운드HR에 자동 접속할 수 없습니다.
- **Cursor**: 지원자 데이터를 읽고 `config/hiring_rubric_2026.md`에 맞춰 등급·근거를 씁니다.
- **message.md**: 전송 전에 사람이 확인·수정할 수 있는 파일입니다.
- **send_slack.bat**: `message.md` 내용을 Slack으로 보냅니다. **토큰은 파일에 저장하지 않고**, 실행할 때 `xoxb-...|C...` 형식으로만 넘깁니다.

## 등급 (간단)

| 등급 | 뜻 |
|------|----|
| **S** | 매우 강한 매치, 즉시 후속 권장 |
| **A** | 서류 통과 권장 |
| **B** | 조건부·추가 검토 |
| **C** | 비권장 (최종은 담당자) |

자세한 배점은 `config/hiring_rubric_2026.md`의 포지션별 표를 따릅니다.

## 매일 필요한 것

- **Cursor** + 라운드HR MCP 연결
- 이 프로젝트 폴더 (`automations/recruitment_slack`)
- Slack 봇이 대상 채널에 초대되어 있을 것
