# 03. 메시지 수정하기 (`message.md` 손으로 고치기)

Cursor가 정리해 준 결과가 마음에 안 들거나, 담당자 판단으로 **문구를 다듬거나 특정 지원자를 빼고 싶을 때** 쓰는 가이드입니다.

## 언제 고치나요?

- 민감한 개인정보(주소·연락처·이름 일부)가 들어간 경우 → 제거
- 근거 문장이 너무 길거나 짧은 경우 → 정리
- 테스트 전송이라서 상단에 `(테스트)` 같은 문구를 넣고 싶은 경우
- C등급은 Slack에 안 보내고 싶을 때 → 해당 블록 통째로 지우기

## 파일 위치

`automations/recruitment_slack/message.md`

## 기본 규칙

- **Slack 마크다운** 형식입니다. HTML이나 복잡한 문법은 쓸 수 없어요.
- 굵게: `*텍스트*`
- 기울임: `_텍스트_`
- 코드 조각: `` `텍스트` ``
- 줄바꿈은 그냥 엔터로 OK

## 양식 예시

```
:briefcase: *서류 단계 지원자 AI 추천 요약*
_기간_: 2026-04-17 09:50 ~ 2026-04-18 09:50 KST
_대상_: `applied` 3명 / 공고 2건

*경영지원실* · *CFO(Chief Financial Officer)* (`job_id=4664`)
_요약_: 여기에 공고별 한 줄 요약

*S* · 홍길동 (id `90001`)
  - 추천·평가: 왜 S인지 한두 문장
  - 리스크: 있으면 한 줄, 없으면 이 줄 지우기

---

*제품* · *서비스 기획자* (`job_id=6001`)
_요약_: ...

*A* · 박기획 (id `90003`)
  - 추천·평가: ...
```

## 자주 하는 편집

### 1) 한 명만 지우기

해당 지원자의 **3줄 블록**(등급·근거·리스크)만 통째로 삭제.

```
*C* · 김지원 (id `90002`)
  - 추천·평가: ...
  - 리스크: ...
```

그리고 상단 `_대상_: ... 명` 숫자를 줄여 두면 더 깔끔합니다.

### 2) 등급만 바꾸기

`*S*` 같은 부분의 알파벳만 `*A*` 등으로 교체. 순서(S→A→B→C)가 맞게 정렬되어 있는지 확인.

### 3) 공고 블록 전체 지우기

`*팀* · *공고*` 줄부터 다음 `---` 줄까지 전부 삭제. 상단 `공고 N건` 숫자도 맞춰 줄이면 더 정확합니다.

### 4) 전체 제목 바꾸기

첫 줄을 원하는 문구로 교체. 예:

```
:briefcase: *2026/04/18 서류 1차 심사 결과*
```

## 저장 후 할 일

1. `Ctrl + S` 로 `message.md` 저장
2. `post_message.bat` 더블클릭
3. Slack 확인

## 실수로 다 지웠어요!

- 양식 복원: `message.template.md` 를 열어 내용을 복사해서 `message.md` 에 다시 붙여넣기.
- 어제 보낸 메시지 복원: Slack에서 어제 메시지를 복사해서 다시 붙여 넣어도 됩니다.
